# Specification

This directory is reserved for GraphTruth's future normative protocol specification: concepts, invariants, canonical representations, versioning, extensions, migrations, and conformance requirements. No normative specification document exists here yet; this README is a non-normative directory notice.

Once published, a versioned specification will define required meaning and behavior without depending on the default implementation, storage engine, model provider, or programming language. Terms such as **MUST**, **SHOULD**, and **MAY** will be normative only in specification documents that explicitly adopt their conventional requirements meaning.

Until then, accepted RFCs record deliberate project decisions, while vision and architecture documents explain intent. After a versioned specification exists, the specification is authoritative for its declared protocol version. An accepted RFC records a decision but does not amend a released specification until incorporated into a new specification version. Schemas and conformance fixtures encode selected requirements and must agree with the applicable specification; implementations have the lowest authority. Any disagreement is a defect to resolve explicitly.

GraphTruth is currently pre-alpha. No compatibility promise exists until versioning and conformance rules say otherwise.
